# Портфолио — Татьяна Кульгейко (RU/EN)

Статичный сайт-портфолио на Vite. Русская версия в корне, английская в `/en/`.

---

## Документация по страницам
Кроме этого README, в `docs/` лежат отдельные файлы — один на главную, по
одному на каждую страницу кейса и один общий на все страницы визуалов. Они
**дополняют** этот README, а не дублируют его: инструкции по запуску/сборке/
деплою — только здесь, а в `docs/*.md` — структура конкретной страницы по
секциям, её уникальные компоненты и **полный список анимаций на странице**
(где и что анимировано, чтобы новую анимацию можно было добавить осознанно,
не пересобирая контекст с нуля по HTML/CSS). Открывай их вместо чтения
исходного HTML кейса — там инлайновые SVG-схемы, из-за которых сам файл
очень тяжёлый для беглого просмотра.

- [`docs/homepage.md`](docs/homepage.md) — главная (RU+EN): структура по
  секциям и полный список анимаций, включая scroll-stack кейсов
- [`docs/samokat-geo.md`](docs/samokat-geo.md) — «География в конструкторе акций»
- [`docs/beeline-crm.md`](docs/beeline-crm.md) — «CRM для операторов колл-центра»
- [`docs/fortis-drivers.md`](docs/fortis-drivers.md) — «Приложение для дальнобойщиков»
- [`docs/fortis-ds.md`](docs/fortis-ds.md) — «Дизайн-система логистической компании»
- [`docs/visuals.md`](docs/visuals.md) — все 5 страниц визуалов (общий файл,
  у них одна система на всех)

## С чего начать (первые шаги)

### 1. Установи Node.js
Нужна версия 20 (зафиксирована в `.nvmrc`). Проверь:
```bash
node -v      # должно быть v20.x
```
Если нет — поставь с nodejs.org (LTS) или через nvm: `nvm use`.

### 2. Установи зависимости (один раз)
```bash
npm install
```

### 3. Запусти локально
```bash
npm run dev
```
Откроется `http://localhost:5173`. Правишь файлы — страница обновляется сама.
- Русская: `http://localhost:5173/`
- Английская: `http://localhost:5173/en/`

> **Важно:** словари `src/locales/*.json` и данные `src/data/*.json`
> читаются в `vite.config.js` один раз при старте сервера (`readFileSync`,
> не per-request). Правки CSS/HTML подхватываются на лету, а вот изменения
> в этих JSON — только после перезапуска `npm run dev` (Ctrl+C и заново).

### 4. Собери прод-версию (когда готово)
```bash
npm run build      # результат в папке dist/
npm run preview    # посмотреть собранное локально
```

> Если стоит более новая версия Node (например 22/24), а не 20 из
> `.nvmrc` — на Vite 5 это обычно всё равно работает без проблем.

## Что уже готово
- Каркас проекта: 20 страниц (главная + 4 кейса + 5 визуалов ×2 языка)
- **Главная полностью свёрстана по Figma** (RU и EN): hero, список кейсов,
  таймлайн опыта, сетка визуалов, CTA-блок «Запускаю новые продукты» — все
  цвета/шрифты/отступы сверены с макетом
- **Hero финализирован**: фото — правая половина экрана с обычным отступом
  контейнера (как везде), верх выровнен по верху текста, высота и текстовый
  блок подстроены под первый экран (`100dvh`), после секции — 140px белого
  пространства (64px на мобиле, там своя раскладка через `display:contents`,
  см. комментарии в `hero.css`). Заголовок собирается из массива
  `hero.titleLines` (а не одной строки) и анимируется построчно — каждая
  строка выезжает из-под маски по очереди. H1-типографика вынесена в токены
  `--fs-h1`/`--ls-h1` (56px/-5% десктоп, 32px/-3% мобайл) и переиспользуется
  в заголовках кейсов, разделов и контактов
- **Навигация (header + мобильное меню) финализирована**: якорное меню
  стартует от середины хедера, hover/активное состояние пунктов, лого
  Geist Bold 20px со своей иконкой (она же — favicon), кнопка «Резюме»
  со стрелкой и реальной ссылкой на файл, мобильное меню — плавающая
  тёмная панель с блюром фона, брейкпоинт переключения на мобильную
  версию — 1330px (свой, отдельный от брейкпоинта страниц кейсов, см. ниже).
  Все статические SVG-иконки инлайновые (без внешних файлов/библиотек)
- Переключатель языка, футер — как партиалы
- Контент главной (кейсы, опыт, визуалы) вынесен в данные:
  `src/data/{cases,experience,visuals}.{ru,en}.json` — секции рендерятся
  из них через `{{#each}}`, тексты добавлять/менять только там
- Анимации появления при скролле (CSS + IntersectionObserver);
  подсветка активного пункта меню — через `src/js/active-nav.js`
- **Список кейсов на главной — scroll-stack** (только десктоп ≥901px):
  карточки залипают под хедером и накладываются друг на друга по мере
  скролла (чистый CSS, `position:sticky` + z-index по порядку), плюс лёгкое
  плавное замедление скролла в момент наезда одной карточки на другую
  (`case-stack-scroll.js`). Построчная «шторка» hero теперь есть и в финале
  («Запускаю новые продукты…»). Полная механика и полный список анимаций
  главной — в `docs/homepage.md`, тут не дублирую
- Аналитика подключена по-настоящему, не заготовка: Vercel Analytics
  (`@vercel/analytics`, инициализируется в `main.js`, начинает считать
  сразу после деплоя на Vercel) + кастомные клики через `data-track`
  (`src/js/analytics/events.js`); Clarity встроен, но ждёт свой project ID
  (см. «Перед публикацией»)
- SEO-файлы: sitemap, robots, мета-теги, og-превью (сам файл превью —
  см. «Перед публикацией», его ещё нет)
- Единая шкала скруглений: 4px на всех картинках сайта (hero, кейсы на
  главной, скриншоты внутри кейсов, обложки «Другие кейсы») и на кнопках
  (`.btn`, включая `--outline`) — правило лежит на базовых классах, новым
  картинкам/кнопкам ничего специально включать не нужно
- **Dot-matrix лоадер поверх картинок, пока они грузятся** (идея —
  dotmatrix.zzzzshawn.cloud, переписана на ванильном CSS/JS — на сайте
  нет React/Tailwind/shadcn): `src/styles/components/loader.css` +
  `src/js/image-loader.js`. Точки вставляются JS-ом в 5×5-сетку, порядок —
  спираль по часовой от края к центру (`--dmx-order` на каждой точке).
  Вешается на `<img>` внутри `.case-hero__media`/`.case-shot`/`.case-flow`
  (все три — на страницах кейсов); если картинка уже в кэше (`img.complete`)
  или грузится быстрее 200мс — лоадер вообще не появляется, чтобы не
  мелькать на обычных быстрых загрузках (было: мелькал на весь экран на
  hero-фото). Новым картинкам вне этих трёх классов лоадер сам не
  подключится — нужно либо добавить класс в `HOST_SELECTOR`/`SELECTOR`
  в `image-loader.js`, либо обернуть в один из существующих.
- **Все 4 страницы кейсов свёрстаны на RU** (`samokat-geo` — «География»,
  `beeline-crm` — CRM, `fortis-drivers` — приложение для дальнобойщиков,
  `fortis-ds` — дизайн-система логистической компании): hero, 5–6
  пронумерованных секций (заголовок/подзаголовки/текст в две колонки/
  списки/скриншоты), тёмный блок карточек-итогов, футер «Другие кейсы».
  Картинки везде реальные (`public/images/ru/cases/...`), плейсхолдеров
  не осталось нигде — `.case-shot__placeholder` (серый блок-заглушка)
  остаётся в CSS про запас на случай будущего кейса, но сейчас не
  используется ни на одной странице.
  Общая система для всех кейсов лежит в `src/styles/pages/case.css` +
  `src/partials/case-header.html`/`case-footer.html`:
  - фон-тинт и фон тёмного блока — per-кейс через CSS-переменные
    (`--case-tint-bg`, `--case-dark-bg`, `--case-hero-bg`, `--case-accent`),
    задаются инлайн на `<main class="case-page" style="...">`, так что
    новый кейс с другими цветами не трогает уже готовые страницы;
  - текстовые блоки — модификаторы `--a`/`--b` (левая/правая колонка),
    `--wide` (на всю ширину), `--group` (вторая и следующая пара колонок
    внутри одной секции — с бо́льшим отступом сверху), необязательный
    `.case-section__subheading` над абзацем; `.case-section__text-group-columns`
    также умеет пару «картинка + текст» (не только текст + текст) —
    добавлено для «Спецификации»/«Редактура» в fortis-ds;
  - `.case-section__anchor` — текстовая ссылка-якорь со стрелкой (напр.
    «Сразу к результатам»), скроллит к другому разделу той же страницы;
  - стрелка «назад» в хедере кейса ведёт на конкретную карточку кейса на
    главной (`/#id-кейса`), а не просто в раздел кейсов — это
    `currentCaseId`, который нужно передавать в `{{> case-header }}` и
    `{{> case-footer }}` при подключении партиалов на новой странице;
  - заголовки карточек в футере «Другие кейсы» (`case-footer.html`) — по
    умолчанию просто `{{ this.title }}`, но у части кейсов перенос строки
    зашит вручную через `{{#if (eq this.id "...")}}` + `<br class="other-case__break">`
    (RU-заголовок узкий на десктопе — 1/3 сетки, — и без принудительного
    переноса иногда переносился не по границе фразы, из-за чего карточка
    «прыгала» по высоте относительно соседних). `.other-case__break`
    скрывается на мобиле (`other-cases.css`, брейкпоинт 1100px) — там
    карточка на всю ширину, перенос не нужен. Если у нового кейса
    заголовок в футере переносится некрасиво — тот же приём, для EN
    обычно не нужен (короче/переносится иначе), поэтому ветка EN просто
    выводит `{{ this.title }}`;
  - **брейкпоинт страниц кейсов (и их футера «Другие кейсы») — 1100px**,
    один порог на всё сразу (текст, отступы, H1, картинки). Раньше было
    два разных порога (1100 и 900) — из-за этого около 1024px получался
    гибрид: текст уже мобильный, а отступы/картинки ещё десктопные.
    Мобильный кроп у всех `<picture><source media="...">` в кейсах
    подгружается на том же 1100px, чтобы не было рассинхрона с раскладкой.
    Это НЕ трогает общий сайтовый брейкпоинт хедера/меню (1330px) и
    брейкпоинт остальных секций главной (900px, см. ниже) — они отдельные;
  - анимации: `reveal`/`reveal--fade`/`reveal--up` (+ ступенчатый
    `data-delay`) — базовый набор на hero и карточках-итогах везде. У каждого
    кейса есть и точечные уникальные компоненты/анимации (флоу-схемы и
    развилка сценариев у Географии, баблы-подсказки у водителей Fortis,
    подложка результатов у дизайн-системы Fortis, и т.д.) — **полный список
    по каждой странице, с классами и файлами** — в `docs/<id-кейса>.md`
    (см. «Документация по страницам» в начале файла), тут не дублирую.
- **Все 5 страниц визуалов готовы, RU и EN** (`visuals/visual-1..5.html` +
  `en/visuals/visual-1..5.html`) — единственный раздел сайта, где EN не
  заглушка. У визуалов своя навигация (не сайтовый header/footer): точки
  сверху для перехода к любому визуалу, стрелки снизу для переключения по
  кругу + стрелки клавиатуры (`src/js/visual-key-nav.js`), крестик — закрыть
  и вернуться на главную в `#visuals`. У каждого визуала своя тёмная или
  светлая тема (фон всей страницы + цвет иконок навигации). Полная структура,
  таблица «визуал → тема → заголовок → картинки» и список анимаций —
  в `docs/visuals.md`, тут не дублирую.

## Как устроены языки
- Тексты — в словарях `src/locales/ru.json` и `en.json` (ключ → перевод).
- В разметке — `{{ t.[ключ] }}`, плагин подставляет нужный язык при сборке.
- **Правило:** добавляя новый текст, клади ключ СРАЗУ в оба словаря.
- Картинки: общие → `public/images/shared/`; различающиеся по языку →
  `public/images/ru/…` и `public/images/en/…`.
- Переключатель языка ведёт на парную страницу (тот же кейс на другом языке).

## Перед публикацией не забыть
1. Заменить `example.com` на реальный домен в: `public/sitemap.xml`,
   `public/robots.txt`, `src/partials/head-meta.html` (hreflang).
2. Вставить свой Clarity ID в `src/partials/head-analytics.html`
   (сейчас там плейсхолдер `CLARITY_PROJECT_ID`).
3. Положить og-превью: `public/images/{ru,en}/og-image.jpg` — сейчас
   файлов нет физически, хотя мета-теги на них уже ссылаются.
4. Положить `public/apple-touch-icon.png` — на него тоже уже ссылается
   `head-meta.html`, но файла в `public/` нет.
5. Собрать EN-версии всех 4 кейсов вместо заглушек («страница в разработке»)
   — см. «Что дальше по контенту» выше (визуалы EN уже готовы, этот пункт
   их больше не касается).
6. Вернуть переключатель языка на главной (убрать `hideLangSwitch=true`)
   — см. «Что дальше по контенту».

Шрифты (`.woff2`) уже лежат в `public/fonts/` и подключены — этот пункт
больше не актуален.

## Деплой (публичная ссылка)
Рекомендуется **Vercel**:
1. Залей проект в репозиторий на GitHub.
2. На vercel.com → New Project → выбери репозиторий.
3. Framework preset: **Vite**. Build: `npm run build`, output: `dist`.
4. Deploy. Получишь ссылку вида `имя.vercel.app` (свой домен — в настройках).
5. В дашборде Vercel включи **Web Analytics** (тумблер) — цифры пойдут сразу.

## Структура (кратко)
```
docs/                    по .md на каждый кейс + один общий на визуалы —
                         структура/уникальные компоненты/анимации, см. выше
index.html / en/         страницы (RU в корне, EN в /en)
cases/                   4 страницы кейсов — все готовы на RU
en/cases/                те же 4 кейса на EN — пока заглушки
visuals/ · en/visuals/   5 страниц визуалов ×2 языка — готовы оба языка
src/locales/             словари переводов ru/en
src/data/                контент кейсов/опыта по языкам
src/styles/              CSS: tokens, base, components, sections, animations,
                         pages/case.css — общая система для страниц кейсов,
                         pages/visual.css — общая система для страниц визуалов
src/js/                  main + модули (меню, якоря, язык, анимации, аналитика,
                         visual-key-nav — клавиатурная навигация визуалов)
src/partials/            header, mobile-menu, footer, meta, analytics,
                         case-header/case-footer — хедер/футер страниц кейсов,
                         visual-nav/visual-footer — хедер/футер страниц визуалов
public/                  картинки, шрифты, favicon, sitemap, robots
vite.config.js           входы всех 20 страниц + i18n-контекст партиалов
                          (там же грузятся src/data/*.json в контекст шаблонов)
```
