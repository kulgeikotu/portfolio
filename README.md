# Портфолио — Татьяна Кульгейко (RU/EN)

Статичный сайт-портфолио на Vite. Русская версия в корне, английская в `/en/`.

---

## С чего начать (первые шаги)

### 1. Установи Node.js
Нужна версия 20 (зафиксирована в `.nvmrc`). Проверь:
```bash
node -v      # должно быть v20.x
```
Если нет — поставь с nodejs.org (LTS) или через nvm: `nvm use`.

### 2. Установи зависимости (один раз)
```bash
npm install
```

### 3. Запусти локально
```bash
npm run dev
```
Откроется `http://localhost:5173`. Правишь файлы — страница обновляется сама.
- Русская: `http://localhost:5173/`
- Английская: `http://localhost:5173/en/`

> **Важно:** словари `src/locales/*.json` и данные `src/data/*.json`
> читаются в `vite.config.js` один раз при старте сервера (`readFileSync`,
> не per-request). Правки CSS/HTML подхватываются на лету, а вот изменения
> в этих JSON — только после перезапуска `npm run dev` (Ctrl+C и заново).

### 4. Собери прод-версию (когда готово)
```bash
npm run build      # результат в папке dist/
npm run preview    # посмотреть собранное локально
```

> Если стоит более новая версия Node (например 22/24), а не 20 из
> `.nvmrc` — на Vite 5 это обычно всё равно работает без проблем.

## Что уже готово
- Каркас проекта: 20 страниц (главная + 4 кейса + 5 визуалов ×2 языка)
- **Главная полностью свёрстана по Figma** (RU и EN): hero, список кейсов,
  таймлайн опыта, сетка визуалов, CTA-блок «Запускаю новые продукты» — все
  цвета/шрифты/отступы сверены с макетом
- **Hero финализирован**: фото — правая половина экрана с обычным отступом
  контейнера (как везде), верх выровнен по верху текста, высота и текстовый
  блок подстроены под первый экран (`100dvh`), после секции — 140px белого
  пространства (64px на мобиле, там своя раскладка через `display:contents`,
  см. комментарии в `hero.css`). Заголовок собирается из массива
  `hero.titleLines` (а не одной строки) и анимируется построчно — каждая
  строка выезжает из-под маски по очереди. H1-типографика вынесена в токены
  `--fs-h1`/`--ls-h1` (56px/-5% десктоп, 32px/-3% мобайл) и переиспользуется
  в заголовках кейсов, разделов и контактов
- **Навигация (header + мобильное меню) финализирована**: якорное меню
  стартует от середины хедера, hover/активное состояние пунктов, лого
  Geist Bold 20px со своей иконкой (она же — favicon), кнопка «Резюме»
  со стрелкой и реальной ссылкой на файл, мобильное меню — плавающая
  тёмная панель с блюром фона, брейкпоинт переключения на мобильную
  версию — 1330px. Все статические SVG-иконки инлайновые (без внешних
  файлов/библиотек)
- Переключатель языка, футер — как партиалы
- Контент главной (кейсы, опыт, визуалы) вынесен в данные:
  `src/data/{cases,experience,visuals}.{ru,en}.json` — секции рендерятся
  из них через `{{#each}}`, тексты добавлять/менять только там
- Анимации появления при скролле (CSS + IntersectionObserver);
  подсветка активного пункта меню — через `src/js/active-nav.js`
- Заготовки под аналитику (Vercel + Clarity)
- SEO-файлы: sitemap, robots, мета-теги, og-превью

- **Две страницы кейсов свёрстаны на RU** (`cases/samokat-geo.html` —
  «География», `cases/beeline-crm.html` — CRM): hero, 5–6 пронумерованных
  секций (заголовок/подзаголовки/текст в две колонки/списки/скриншоты),
  тёмный блок карточек-итогов, футер «Другие кейсы». Общая система для
  всех будущих кейсов лежит в `src/styles/pages/case.css` +
  `src/partials/case-header.html`/`case-footer.html`:
  - фон-тинт и фон тёмного блока — per-кейс через CSS-переменные
    (`--case-tint-bg`, `--case-dark-bg`, `--case-hero-bg`, `--case-accent`),
    задаются инлайн на `<main class="case-page" style="...">`, так что
    новый кейс с другими цветами не трогает уже готовые страницы;
  - текстовые блоки — модификаторы `--a`/`--b` (левая/правая колонка),
    `--wide` (на всю ширину), `--group` (вторая и следующая пара колонок
    внутри одной секции — с бо́льшим отступом сверху), необязательный
    `.case-section__subheading` над абзацем;
  - `.case-shot__placeholder` — серый блок-заглушка на месте картинки,
    пока реального скриншота нет (компонент остаётся для будущих кейсов —
    в CRM все картинки уже реальные, см. ниже);
  - `.case-section__anchor` — текстовая ссылка-якорь со стрелкой (напр.
    «Сразу к результатам»), скроллит к другому разделу той же страницы;
  - стрелка «назад» в хедере кейса ведёт на конкретную карточку кейса на
    главной (`/#id-кейса`), а не просто в раздел кейсов — это
    `currentCaseId`, который нужно передавать в `{{> case-header }}` и
    `{{> case-footer }}` при подключении партиалов на новой странице.
  - анимации: `reveal`/`reveal--fade`/`reveal--up` (+ ступенчатый
    `data-delay`) — базовый набор на hero и карточках-итогах; для
    Географии дополнительно — покадровое появление шагов флоу-схем
    (`.flow-step`, инлайн-SVG), световой блик по цитате (`.case-quote`),
    блик и дорисовка галочки на карточках-итогах, курсор-параллакс на
    обложках (переиспользует `case-parallax.js` с главной). У CRM своих
    step-анимаций на схемах нет (там обычные статичные SVG/картинки) —
    можно добавить по тому же принципу, что у Географии, если понадобится.
- **Картинки CRM — все реальные** (`public/images/ru/cases/beeline-crm/` +
  `stats-bg.svg` для тёмного блока), плейсхолдеров не осталось. Обложка
  Географии для футера «Другие кейсы» тоже на месте
  (`public/images/shared/cases/previews/samokat-geo.webp` — раньше её не
  было в данных вообще, из-за этого обложка была битой).
- `docs/case-brief-template.md` — чистый шаблон брифа под новый кейс,
  обновлён по опыту CRM (явно просит помечать паттерн заголовков —

## Как устроены языки
- Тексты — в словарях `src/locales/ru.json` и `en.json` (ключ → перевод).
- В разметке — `{{ t.[ключ] }}`, плагин подставляет нужный язык при сборке.
- **Правило:** добавляя новый текст, клади ключ СРАЗУ в оба словаря.
- Картинки: общие → `public/images/shared/`; различающиеся по языку →
  `public/images/ru/…` и `public/images/en/…`.
- Переключатель языка ведёт на парную страницу (тот же кейс на другом языке).

## Перед публикацией не забыть
1. Заменить `example.com` на реальный домен в: `public/sitemap.xml`,
   `public/robots.txt`, `src/partials/head-meta.html` (hreflang).
2. Вставить свой Clarity ID в `src/partials/head-analytics.html`.
3. Положить шрифты `.woff2` в `public/fonts/` (см. `src/styles/fonts.css`).
4. Положить og-превью: `public/images/{ru,en}/og-image.jpg`. Остальные
   картинки (hero, кейсы, визуалы) уже на месте.
5. Собрать оставшиеся внутренние страницы кейсов/визуалов вместо заглушек
   («страница в разработке») — см. «Что дальше по контенту» выше.
6. Решить с блюром NDA-скрина в CRM и вернуть переключатель языка на
   главной (убрать `hideLangSwitch=true`) — см. «Что дальше по контенту».

## Деплой (публичная ссылка)
Рекомендуется **Vercel**:
1. Залей проект в репозиторий на GitHub.
2. На vercel.com → New Project → выбери репозиторий.
3. Framework preset: **Vite**. Build: `npm run build`, output: `dist`.
4. Deploy. Получишь ссылку вида `имя.vercel.app` (свой домен — в настройках).
5. В дашборде Vercel включи **Web Analytics** (тумблер) — цифры пойдут сразу.

## Структура (кратко)
```
index.html / en/         страницы (RU в корне, EN в /en)
cases/ · visuals/        внутренние страницы (2 кейса готовы, остальное — заглушки)
docs/                    бриф-шаблон под новый кейс + рабочие референсы/скрины
src/locales/             словари переводов ru/en
src/data/                контент кейсов/опыта по языкам
src/styles/              CSS: tokens, base, components, sections, animations,
                         pages/case.css — общая система для страниц кейсов
src/js/                  main + модули (меню, якоря, язык, анимации, аналитика)
src/partials/            header, mobile-menu, footer, meta, analytics,
                         case-header/case-footer — хедер/футер страниц кейсов
public/                  картинки, шрифты, favicon, sitemap, robots
vite.config.js           входы всех 20 страниц + i18n-контекст партиалов
                          (там же грузятся src/data/*.json в контекст шаблонов)
```
