- id: `beeline-crm`

## Структура

**Hero** — вариант с KPI над подзаголовком (`.case-hero__callout` +
`.case-hero__stats`, 2 цифры: «146 виджетов», «1000+ пользователей»), а не
просто `.case-hero__subtitle` как в Географии. H1 «CRM для операторов<br>колл-центра».

**01 Контекст и проблема** — A: подзаголовок «Продукт» + текст о продукте.
B: подзаголовок «Проблема» + текст + список проблем легаси-системы. Медиа:
`.case-flow` — блок-схема вопросов одной из диагностик (`context-diagnostics-flow.webp`).

**02 Задача** — A: текст задачи. B: текст + список. Медиа: `.case-flow`
(`task-flow.svg`), без figcaption.

**03 Мой вклад** — A: подзаголовок «Основной вызов». B: подзаголовок «Ключевые
действия» + список. Внизу колонки B — `.case-section__anchor--bottom` со
стрелкой «Сразу к результатам», скроллит к `#results` (секция 06).

**04 Исследования** — `.case-section__text-group` «Кабинетное»: пара колонок
(список слева, текст справа) через `.case-section__text-group-columns`. Ниже —
вторая пара как `--group` (больший отступ сверху): «Интервью» / «Некоторые из
инсайтов». Медиа: `.case-shot` — сториборд по итогам интервью новенького
(`research-storyboard.webp`).

**05 Решение** — A: «Информационная архитектура» + список. B: «Подход» + текст.
Медиа: `.case-flow` — набросок low-fi (`solution-lowfi.svg`, figcaption «Один
из набросков low-fi»). Ниже — вторая пара как `--group`: «Сценарии» / «Валидация»,
у каждой свой список. Два `.case-shot` медиа: результаты UX-тестов
(`solution-ux-test.webp`) и «Ключевые экраны под NDA» (`solution-key-screens.webp`,
**сейчас без блюра** — открытый вопрос перед публикацией, см. README «Что
дальше по контенту»).

**06 Запуск и результаты** (`id="results"`) — `.case-section__text-group`
«Разработка» (текст + `.case-flow` — `launch-flow.svg`) и «Пилот» (текст +
список) как вторая группа `--group`.

**Тёмный блок (итоги)** — `.case-stats` (обычная, НЕ `--triple`), 4 карточки
2×2: пересобрала legacy-сценарий без сопротивления операторов · провела
исследования, повлияла на бэклог · декомпозировала задачи, подключила
junior-дизайнера · перевела продукт на дизайн-систему.

**Футер** — `{{> case-footer currentCaseId="beeline-crm" }}`.

## Уникальные компоненты этой страницы
Ничего сверх общей системы — самый «базовый» из четырёх кейсов по компонентам
(README называет это явно: «у CRM и дизайн-системы Fortis — только базовый
набор, без уникальных компонентов»). Единственная местная особенность —
плотное использование `.case-section__text-group`/`.case-section__text-group-columns`
для парных подзаголовков (Кабинетное/Интервью, Разработка/Пилот) — паттерн
общий (см. README), но нигде на сайте не используется так часто.

## Анимации на этой странице
| Где | Элемент/класс | Эффект | Файл |
|---|---|---|---|
| Hero | `h1.reveal`, `.case-hero__callout.reveal--fade`, `.case-hero__media.reveal--fade` | появление при скролле (база) | `animations.css` + `reveal-on-scroll.js` |
| Тёмный блок | `.case-stats__item.reveal--up[data-delay="0..3"]` ×4 | появление снизу, шаг 0.08s | `animations.css` |
| Тёмный блок | `.case-stats__item::after` | один раз, блик-шиммер по стеклу карточки | `case.css` (`@keyframes case-stats-shimmer`) |
| Тёмный блок | `.case-stats__icon path` | один раз, галочка дорисовывается (`stroke-dashoffset`) | `case.css` (`@keyframes case-stats-check`) |
| Футер «Другие кейсы» | `.other-case__cover` (обложки 3 других кейсов) | курсор-параллакс при hover (десктоп) | `case-parallax.js` |

Никаких уникальных для этой страницы анимаций (в отличие от Географии/Fortis-
водителей/Fortis-DS) — только базовый `reveal` набор + общий для всех кейсов
тёмный блок итогов. Если будешь добавлять анимацию именно на эту страницу —
свободных «крючков» под конкретные секции (01-06) сейчас нет, потребуется
завести новый класс по аналогии с `.case-quote`/`.case-decision`/`.case-callouts`
у других кейсов.
